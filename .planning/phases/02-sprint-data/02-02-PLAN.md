**Sections:**

`objective`: Fetch and display the current sprint backlog from Azure DevOps. Show sprint metadata (name, dates, iteration path) followed by work items grouped by parent story.

`execution_context`: Reference the helper script and config file paths.

`context`: Document the CLI contracts for `get-sprint` and `get-sprint-items` (input/output/exit codes). Include $CWD explanation.

`process`: Define these steps:

1. **Check prerequisites:** Verify `~/.claude/get-shit-done/bin/azdo-tools.cjs` exists. If not, tell user to check GSD installation. Also check config exists by running `node ~/.claude/get-shit-done/bin/azdo-tools.cjs load-config --cwd $CWD`. If exit 1, tell user to run `/gsd:azdo-setup` first.

2. **Fetch sprint metadata:** Run `node ~/.claude/get-shit-done/bin/azdo-tools.cjs get-sprint --cwd $CWD`. If exit 1, show error from stderr and stop. Parse JSON from stdout.

3. **Fetch sprint items:** Run `node ~/.claude/get-shit-done/bin/azdo-tools.cjs get-sprint-items --cwd $CWD`. If exit 1, show error from stderr and stop. Parse JSON array from stdout.

4. **Format and display:** Present the data in this exact terminal format:

```
=== Sprint: {name} ===
Iteration: {path}
Dates: {startDate formatted as YYYY-MM-DD} to {finishDate formatted as YYYY-MM-DD}
Items: {total count}

---

[{type abbreviation}] #{id} -- {title} ({state})
     {description, first 3 lines only, indented 5 spaces}
     Acceptance Criteria:
       {each criterion on its own line, indented 7 spaces}
     Tasks:
       [{type}] #{id} -- {title} ({state})
       [{type}] #{id} -- {title} ({state})

[{type abbreviation}] #{id} -- {title} ({state})
     (no description)
     (no acceptance criteria)
     Tasks:
       (none)
```

**Formatting rules:**
- Type abbreviations: "User Story" -> "US", "Task" -> "Task", "Bug" -> "Bug", all others use full type name
- Group items by parent: items with no parentId are "top-level" (stories/bugs). Items with a parentId are children (tasks) displayed under their parent.
- If a child references a parentId that is NOT in the sprint items list, display it as a top-level item instead (orphaned task)
- Description: show first 3 lines only. If longer, append "..." on line 4. If empty, show "(no description)".
- Acceptance Criteria: If empty, show "(no acceptance criteria)".
- Tasks section under each parent: If no children, show "(none)".
- Dates: Parse ISO date strings, display as YYYY-MM-DD. If null/undefined, show "not set".
- Use plain dashes "--" not em-dashes for the separator between id and title.

5. **Handle edge cases:**
   - Empty sprint (0 items): Display sprint header, then "No work items in this sprint."
   - No active sprint: The get-sprint command already returns an error; display it.

`success_criteria`:
- Sprint name, iteration path, and dates are shown
- Work items are grouped by parent story
- No HTML tags appear in the output
- Output is readable in a standard terminal (no excessive line length, no broken formatting)
