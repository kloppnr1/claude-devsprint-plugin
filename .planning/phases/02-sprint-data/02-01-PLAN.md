**Helper 1: stripHtml(html)**
```javascript
function stripHtml(html) {
  if (!html) return '';
  return html
    .replace(/<br\s*\/?>/gi, '\n')
    .replace(/<\/p>/gi, '\n')
    .replace(/<\/li>/gi, '\n')
    .replace(/<[^>]+>/g, '')
    .replace(/&lt;/g, '<')
    .replace(/&gt;/g, '>')
    .replace(/&amp;/g, '&')
    .replace(/&nbsp;/g, ' ')
    .replace(/&#\d+;/g, '')
    .replace(/\n{3,}/g, '\n\n')
    .trim();
}
```

**Helper 2: resolveTeamName(org, encodedProject, encodedPat)**
- Step 1: Try using `decodeURIComponent(encodedProject)` as the team name. Call `GET {org}/{encodedProject}/{encodeURIComponent(decodedProject)}/_apis/work/teamsettings/iterations?$timeframe=current&api-version=7.1`. If status 200 and response has `value` array, return the decoded project name.
- Step 2 (fallback): Call `GET {org}/_apis/projects/{encodedProject}/teams?$top=1&api-version=7.1`. If status 200 and has teams, return `teams.value[0].name`.
- Step 3 (error): Throw Error with message "Could not resolve team name. Check that the project has at least one team configured."
- Note: The step 1 check does NOT require `value.length > 0` — just getting a 200 with a valid response means the team name works. The `$timeframe=current` filter may return empty values array when no sprint is active, but the team name is still valid.

**Helper 3: getSprintData(cwd)**
Shared logic used by both get-sprint and get-sprint-items:
- Load config via `loadConfig(cwd)`
- Re-encode PAT: `Buffer.from(':' + cfg.pat).toString('base64')`
- Call `resolveTeamName(cfg.org, cfg.project, encodedPat)` to get team name
- Call `GET {org}/{project}/{encodeURIComponent(teamName)}/_apis/work/teamsettings/iterations?$timeframe=current&api-version=7.1`
- Parse response. If `data.value.length === 0`, throw Error "No active sprint found. Check that a sprint with today's date range is active in Azure DevOps."
- Return `{ cfg, encodedPat, teamName, iteration: data.value[0] }` where iteration has `{ id, name, path, attributes: { startDate, finishDate } }`

**Command: cmdGetSprint(cwd)**
- Call `getSprintData(cwd)`
- Output JSON to stdout: `{ iterationId: iteration.id, name: iteration.name, path: iteration.path, startDate: iteration.attributes.startDate, finishDate: iteration.attributes.finishDate }`
- Exit 0 on success, exit 1 on error (wrap in try/catch, errors go to stderr)

   const idSet = new Set();
